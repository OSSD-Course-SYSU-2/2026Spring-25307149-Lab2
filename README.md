# 鸿蒙多端自适应太空射击游戏｜OSSD课程实验Lab2
## 一、项目概述
本项目为操作系统系统开发课程实验作业，基于 HarmonyOS NEXT 6.0.2，使用ArkTS + ArkUI Canvas开发一款2D太空射击休闲游戏。
项目核心目标不仅实现完整射击游戏玩法，同时重点完成**全设备多端自适应适配**，解决鸿蒙手机、平板、模拟器布局错位、触控失效、操作按键消失等兼容性问题，完整展示ArkUI响应式布局、原生Canvas图形绘制、多点触摸事件处理、屏幕尺寸动态适配的开发思路。

## 二、整体设计思路
### 2.1 游戏分层架构设计
采用Stack层级分层渲染思想，界面分为两层，渲染顺序自下而上：
1. **底层游戏渲染层（Canvas画布）**
独占绝大部分屏幕可视区域，负责飞船、敌机、子弹、道具、粒子效果实时绘制；通过帧循环持续刷新画面，完成碰撞检测、游戏数值计算。
2. **上层悬浮交互层（操作控制面板）**
悬浮在画布顶部，包含左侧虚拟摇杆、右侧2×2技能按钮、顶部生命值/得分UI；独立分层不会被游戏画面遮挡，保证操作控件永久可见。

### 2.2 多端自适应核心设计思路
摒弃传统硬编码固定vp尺寸，采用**动态屏幕参数获取+百分比相对布局**方案，实现一套代码适配所有鸿蒙设备：
1. 实时获取设备真实分辨率、窗口安全区（顶部状态栏、底部手势条），自动预留内边距，避免UI被系统界面截断；
2. 通过屏幕宽度阈值自动区分设备形态：手机竖屏紧凑布局、平板横屏宽幅布局，控件尺寸、间距随屏幕宽高等比例缩放；
3. 触摸坐标归一化算法：将设备原生触摸像素坐标转换为0~1区间相对比例，再映射至画布坐标，彻底消除不同分辨率设备触控偏移、拖动失灵问题；
4. 窗口尺寸监听：设备旋转、模拟器窗口缩放时，实时重新计算画布、摇杆、按钮尺寸，界面动态重绘无错乱。

### 2.3 虚拟摇杆与技能按键交互设计思路
1. 左侧虚拟摇杆：圆形底座+可拖拽摇杆头，限定最大拖动偏移阈值，松手自动平滑回弹；仅当拖动偏移超过30%才识别移动指令，降低误触概率；分离上下左右移动状态标记，供飞船帧循环读取移动向量。
2. 右侧技能按钮组：2行2列网格紧凑排布圆形按钮，区分点击单次触发/长按持续释放逻辑；按钮尺寸、间距动态计算，平板自动放大控件，手机缩小节省画面空间。

### 2.4 游戏数值与逻辑解耦设计
将飞船位置、血量、得分、游戏状态、摇杆触控状态全部使用`@State`响应式变量管理，ArkUI自动响应状态变更刷新界面；游戏碰撞、道具生成、子弹运动逻辑独立封装，与布局适配代码分层隔离，便于后续功能迭代与维护。

## 三、核心游戏特色
### 3.1 完整太空射击游戏玩法
1. **飞船移动操控**
左侧虚拟摇杆全方位控制飞船八向移动，设置屏幕边界碰撞限制，飞船不会移出画布可视区域。

2. **四类特色战斗技能**
- 普通射击：持续发射子弹攻击前方敌机；
- 多段散射：一次性发射多枚扇形子弹，扩大攻击范围；
- 全屏清除：清空屏幕所有敌方单位，快速脱困；
- 子弹流转：短时间无冷却持续射击，输出拉满。

3. **随机道具与敌机系统**
画布顶部定时随机生成敌机、钻石增益道具；实现物体碰撞判定：敌机撞击飞船扣除生命值，拾取钻石增加游戏得分。

4. **完整游戏状态体系**
实时左上角爱心生命值UI、右上角得分靶标计数；生命值归零时触发游戏结束状态，锁定摇杆与所有技能按键。

### 3.2 跨设备全兼容自适应（项目最大亮点）
1. 兼容HarmonyOS NEXT 6.0.2真机、多分辨率模拟器、平板横/竖屏；
2. 自动避让系统安全区，彻底解决模拟器调试时摇杆、按钮被挤出屏幕、完全消失的问题；
3. 触控事件全局适配，无论设备分辨率大小，摇杆拖动、技能点击逻辑完全一致，无坐标错位；
4. 纯相对百分比布局，无任何写死固定宽高数值，打包后可一键部署至任意鸿蒙设备，无需单独修改布局代码。

### 3.3 人性化交互优化
1. 摇杆低误触机制：小幅拖动不触发移动，仅大幅度滑动才响应操作，避免手指误碰造成飞船乱飘；
2. 悬浮半透明操作面板，不遮挡核心太空游戏画面；
3. 触摸事件完善生命周期：按下、拖动、松手三阶段完整处理，松手瞬间摇杆自动复位、停止飞船移动；
4. 顶部轻量化UI，生命值与得分图标小巧简洁，占用画面空间极小。

### 3.4 原生ArkUI高性能渲染
基于系统原生Canvas实现实时帧动画，无第三方游戏引擎依赖，轻量低占用；帧循环高效刷新画面，敌机、子弹批量绘制无明显掉帧，适配移动端低性能设备流畅运行。

## 六、运行部署教程
1. 使用DevEco Studio打开项目根目录Suhlin1；
2. 连接鸿蒙真机 / 启动HarmonyOS NEXT 6.0.2模拟器，点亮并解锁设备屏幕；
3. 等待项目同步构建完成，点击顶部绿色运行按钮；
4. 设备自动安装应用，进入游戏后左侧摇杆控制飞船移动，点击右侧按钮释放对应战斗技能。

## 七、迭代优化记录
1. 初始版本：固定vp布局，模拟器下操作按键超出屏幕、触控坐标偏移；
2. 迭代1：新增屏幕动态尺寸获取、安全区避让，解决按键消失问题；
3. 迭代2：重构触摸归一化逻辑，修复多设备触控错位；
4. 迭代3：区分手机/平板自适应布局，缩放摇杆与按钮尺寸，降低误触；
5. 迭代4：完善游戏完整逻辑，新增多类型技能、道具、生命值计分系统。
